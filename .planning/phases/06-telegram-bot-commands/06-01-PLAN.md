Create TranscriptParser.swift porting the TypeScript transcript-parser.ts logic to Swift. This is a pure-function module (enum with static methods, no state).

**Types to define:**

1. `SessionEvent` struct: type (String), sessionId (String), uuid (String), timestamp (String), cwd (String?), gitBranch (String?), slug (String?), message (MessageContent?), isMeta (Bool), data (JSONValue?)
2. `MessageContent` struct: role (String), content (MessageContentValue) where MessageContentValue is an enum of .string(String) or .array([ContentBlock])
3. `ContentBlock` struct: type (String), text (String?), name (String?), content (String?)
4. `SessionSummary` struct: sessionId, slug, cwd, gitBranch, startTime, lastActivity, turns ([ConversationTurn]), toolCalls (Int), errors (Int), lastUserPrompt (String), lastPairedResponse (String)

**Functions to implement:**

1. `parseTranscript(filePath: String) -> [SessionEvent]` — Read file line-by-line, JSON decode each line, skip empty lines and parse failures (log warning, don't crash)
2. `extractSessionSummary(events: [SessionEvent]) -> SessionSummary` — Port the TS logic:
   - Filter user messages: type=="user", role=="user", text length >10, not system noise
   - System noise patterns: strings containing "&lt;command-name&gt;", "&lt;system-reminder&gt;", "&lt;bash-stdout&gt;", "This session is being continued", "Stop hook feedback", "Implement the following plan", "Tool loaded", "[Request interrupted"
   - Count tool calls: assistant events with content blocks where type=="tool_use"
   - Count errors: events where data.type == "error"
   - Build ConversationTurn array: for each real user prompt, find longest assistant text response before next user prompt, collect tool counts as "ToolName x3, OtherTool x2" format
   - Find lastUserPrompt and lastPairedResponse using backwards scan
3. `extractUserText(content: MessageContentValue) -> String` — Handle both string and array content
4. `isSystemNoise(_ text: String) -> Bool` — Check against noise patterns

**Use JSONSerialization (Foundation) for parsing** since JSONL lines are arbitrary JSON structures. Parse each line as [String: Any] dictionary, then extract typed fields manually. Do NOT use Codable — the JSONL schema is too loose/variable.

**Important:** The function produces `[ConversationTurn]` using the EXISTING ConversationTurn struct from SummaryEngine.swift. Do NOT redefine it.

**ClaudeProcess.swift:**

Create a class that manages spawning and streaming the claude CLI.

1. `ClaudeStreamEvent` enum: `.text(String)` for accumulated assistant text, `.toolUse(name: String)` for tool call notifications, `.error(String)` for errors, `.done` for completion
2. `ClaudeProcess` final class (Sendable):
   - Properties: `process: Process`, `outputPipe: Pipe`, `isRunning: Bool`
   - `init()` — no-arg
   - `func run(prompt: String, model: String, cwd: String, resume sessionId: String? = nil, maxTurns: Int = 15) -> AsyncThrowingStream<ClaudeStreamEvent, Error>` — The main entry point:
     a. Create Foundation `Process()`, set `executableURL` to `URL(fileURLWithPath: Config.claudeCliPath)`
     b. Build arguments: `["--print", "--output-format", "stream-json", "-p", prompt, "--model", model, "--max-turns", String(maxTurns)]`
     c. If sessionId is provided for resume (BOT-06): add `["--resume", sessionId]`
     d. Set `process.currentDirectoryURL = URL(fileURLWithPath: cwd)`
     e. **CRITICAL (CLI-03):** Build environment from ProcessInfo.processInfo.environment, then DELETE the "CLAUDECODE" key: `var env = ProcessInfo.processInfo.environment; env.removeValue(forKey: "CLAUDECODE"); process.environment = env`
     f. Set up stdout Pipe, attach to process
     g. Launch process
     h. Return AsyncThrowingStream that reads from the pipe's fileHandleForReading.bytes.lines, parses each line as JSON (NDJSON format), extracts assistant text blocks and tool_use events, yields ClaudeStreamEvent values
   - `func cancel()` — Terminate the process if running (process.terminate())

**NDJSON parsing within the stream:**
Each line is a JSON object. Key event types from claude CLI stream-json output:

- `{"type": "assistant", "message": {"content": [{"type": "text", "text": "..."}]}}` → yield .text with accumulated text
- `{"type": "assistant", "message": {"content": [{"type": "tool_use", "name": "..."}]}}` → yield .toolUse
- `{"type": "result", ...}` → yield .done
- Lines that fail JSON parse → log warning, skip (don't crash)

**Thread safety:** The Process lifecycle is managed within the AsyncThrowingStream closure. The cancel() method calls process.terminate() which is safe from any thread.
